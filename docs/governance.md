# Governance

## Policy

All newly deployed infrastructure must follow this naming standard.

Exceptions require approval by the IT Architecture or DevOps team.

This document set should be reviewed periodically as infrastructure evolves.

## Ownership

- Owner: `IT Architecture / DevOps`
- Status: `Active`
- Version: `1.0`

## Maintenance Guidance

- Keep the naming model stable unless there is a strong operational reason to change it.
- Extend role codes through the shared catalog instead of local conventions.
- Update examples when new approved patterns are introduced.
- Review new role codes through the contribution workflow.
